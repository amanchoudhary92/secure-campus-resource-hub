# Screenshots Checklist

Add screenshots here after deployment for the GitHub README.

Recommended screenshot order:

1. `dashboard.png` - Platform overview, real registered users, resource stats.
2. `resources.png` - Approved resources with search/filter and AI summaries.
3. `upload.png` - Secure upload form and validation messaging.
4. `admin-moderation.png` - Admin approve/reject/block/delete queue.
5. `admin-users.png` - User block/unblock dashboard.
6. `admin-duplicates.png` - Duplicate file hash dashboard.
7. `admin-audit.png` - Security audit log table.
8. `profile-my-uploads.png` - Student profile and upload status tracking.
9. `requests.png` - Login-only resource request board.

## Screenshot safety rules

Before adding screenshots:

- Hide or crop real email addresses if needed.
- Do not show `.env.local`.
- Do not show Supabase service role keys.
- Do not show private admin credentials.
- Do not include sensitive student data.
- Use test/demo uploads where possible.

You can reference screenshots in README later like:

```md
![Dashboard](docs/screenshots/dashboard.png)
```
